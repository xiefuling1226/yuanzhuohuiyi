# 🚀 Gitee Pages 部署指南

## 准备工作

已为你准备好干净的部署文件在 `deploy-gitee` 文件夹中。

---

## 第一步：注册并实名认证

1. 访问 https://gitee.com
2. 注册账号（手机号即可）
3. **完成实名认证**（必须！否则无法开启 Pages）
   - 点击右上角头像 → 设置 → 实名认证
   - 使用支付宝扫码快速认证

---

## 第二步：创建仓库

### 网页操作：

1. 登录后，点击右上角 **"+"** → **"新建仓库"**
2. 填写：
   - **仓库名称**：`roundtable`（或你想要的名字）
   - **是否开源**：✅ 选择 **"公开"**
   - **初始化仓库**：✅ 勾选 **"使用 README 初始化"**
3. 点击 **"创建"**

---

## 第三步：上传代码

### 方法一：网页上传（推荐新手）

1. 进入刚创建的仓库
2. 点击 **"上传文件"** 按钮
3. 打开 `deploy-gitee` 文件夹
4. **全选所有文件**（Cmd+A），拖到网页上传区域
5. 填写提交信息：`初始部署`
6. 点击 **"提交"**

### 方法二：Git 命令行（推荐）

在你的终端中执行：

```bash
# 1. 进入部署文件夹
cd /Users/fulingxie/Qorder/圆桌会议/deploy-gitee

# 2. 初始化 Git 仓库
git init

# 3. 添加所有文件
git add .

# 4. 提交
git commit -m "初始部署"

# 5. 添加远程仓库（替换成你自己的 Gitee 仓库地址）
# 格式：https://gitee.com/你的用户名/仓库名.git
git remote add origin https://gitee.com/YOUR_USERNAME/roundtable.git

# 6. 推送代码
git branch -M main
git push -u origin main
```

**如何获取仓库地址？**
- 在 Gitee 仓库页面，点击 **"克隆/下载"** 按钮
- 复制 HTTPS 地址

---

## 第四步：开启 Gitee Pages

1. 进入你的 Gitee 仓库
2. 点击顶部导航栏的 **"服务"** → **"Gitee Pages"**
3. 配置：
   - **部署分支**：选择 `main`（或 `master`）
   - **部署目录**：留空（默认根目录）
   - **强制使用 HTTPS**：✅ 勾选
4. 点击 **"启动"**
5. 等待 1-2 分钟，部署完成

---

## 第五步：访问你的网站

部署完成后，你会看到一个访问地址，格式为：

```
https://你的用户名.gitee.io/roundtable/
```

**这就是你的在线网站地址！**可以分享给任何人访问。

---

## 后续更新代码

### 网页更新：
1. 进入仓库 → 上传文件 → 覆盖旧文件 → 提交
2. 重新点击 **服务** → **Gitee Pages** → **更新**

### Git 更新：
```bash
cd /Users/fulingxie/Qorder/圆桌会议/deploy-gitee

# 修改文件后...
git add .
git commit -m "更新说明"
git push

# 然后去 Gitee 网页点击"更新" Pages
```

---

## ⚠️ 注意事项

1. **必须实名认证**：否则无法使用 Pages 功能
2. **公开仓库**：免费用户只能部署公开仓库
3. **更新延迟**：代码推送后，需要手动点击"更新" Pages
4. **域名限制**：免费用户只能使用 `gitee.io` 子域名
5. **备案**：如果使用自定义域名，需要 ICP 备案

## 💡 遇到问题？

### Q: 实名认证失败？
A: 使用支付宝扫码认证最快，一般即时通过。

### Q: Pages 启动失败？
A: 检查是否有 `index.html` 文件，必须在根目录。

### Q: 访问 404？
A: 等待 2-3 分钟，Pages 需要时间构建。

### Q: 国内访问慢？
A: Gitee 本身就在国内，应该很快。如果慢，检查网络。
